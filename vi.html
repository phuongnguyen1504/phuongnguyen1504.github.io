CPS – Copro Platform System Hiện tại

Hệ thống ERP nội bộ · Công ty IT Staffing Nhật Bản (SES) · Rikkeisoft

Copro là công ty Nhật Bản chuyên phái cử kỹ sư IT (mô hình SES) — tuyển dụng kỹ sư rồi cử đi làm việc tại các công ty khách hàng theo hợp đồng dịch vụ. CPS là hệ thống quản lý nội bộ giúp nhân viên Copro theo dõi hồ sơ kỹ sư, hợp đồng phái cử, chấm công và xuất hóa đơn gửi khách hàng.

Công việc chính: Phụ trách module xuất tài liệu — viết logic query DB, generate báo cáo Excel/PDF rồi upload lên AWS S3. Trực tiếp triển khai 6 loại tài liệu pháp lý theo chuẩn luật lao động Nhật:

  • 雇用通知書 — Giấy thông báo điều kiện lao động phát cho kỹ sư khi ký hợp đồng mới
  • 労働者派遣個別契約書 — Hợp đồng phái cử chi tiết giữa Copro và công ty khách hàng
  • 労働者派遣通知書 — Thông báo bảo hiểm xã hội khi kỹ sư bắt đầu phái cử
  • 派遣先・派遣元管理台帳 — Sổ lưu hồ sơ phái cử hai phía (bắt buộc theo luật)
  • 年次有給休暇管理簿 — Danh sách theo dõi số ngày phép năm của toàn bộ kỹ sư theo chi nhánh

AI Engineering: Tự xây dựng pipeline: Excel spec nghiệp vụ → tài liệu markdown kỹ thuật → spawn AI coding agent tự viết code. Kết hợp MCP + codegraph để AI agent hiểu kiến trúc hệ thống và sinh execution plan chính xác. Kết quả: giảm thời gian từ nhận task đến test từ 4 ngày xuống còn 1.5 ngày.

System EXE – Database Migration

Oracle → MySQL · 60 bảng · Khách hàng: IDEMITSU (Tập đoàn năng lượng Nhật Bản) · Rikkeisoft

IDEMITSU — tập đoàn năng lượng xăng dầu lớn của Nhật Bản — cần migrate toàn bộ dữ liệu từ Oracle sang MySQL để giảm chi phí vận hành. Tổng cộng 60 tables, trong đó table doanh số bán dầu nhờn theo ngày chứa 100M+ records.

Công việc chính: Thiết kế và triển khai toàn bộ pipeline migration bằng Spring Batch:

  • Spring Batch tasklet + chunk query Oracle, export ra CSV, bổ sung thêm các field từ master table
  • Shell script tự động cắt file CSV lớn (100M+ records) thành các file 100.000 records/file
  • Spring Batch chunk đọc lần lượt từng file CSV nhỏ và import vào MySQL

Vấn đề gặp phải: Ban đầu bị lỗi Java heap khi xử lý file CSV quá lớn. Giải quyết bằng cách tách file thành chunks nhỏ để kiểm soát memory, đảm bảo ổn định trong suốt quá trình migration production.

Schedule Board – Quản lý lịch giảng viên Đại học Chiba

Ứng dụng quản lý lịch · Khách hàng: Chiba University (Nhật Bản) · Toppa Solution

Web app giúp giảng viên Đại học Chiba đăng ký và quản lý lịch họp, đặt phòng họp, đính kèm tài liệu cuộc họp và nhận thông báo qua email.

Công việc chính:

  • Xây dựng tính năng drag & drop trên calendar để sắp xếp lịch họp trực quan
  • Tích hợp gửi email nhắc lịch họp tự động cho giảng viên
  • Viết các API mới theo thiết kế Figma: quản lý phòng họp, tài liệu đính kèm, tìm kiếm lịch

Thách thức kỹ thuật: Xử lý bài toán single-session login qua WebSocket — mỗi tài khoản chỉ được đăng nhập từ một thiết bị tại một thời điểm. Giải pháp:

  • Heartbeat định kỳ để phát hiện và xóa kết nối WebSocket bị ngắt âm thầm (zombie connection)
  • Lưu session theo userId trong Redis, cập nhật atomic khi có đăng nhập mới từ thiết bị khác
  • Gắn deviceId vào mỗi tab trình duyệt để phân biệt multi-tab của cùng một người dùng
  • Fallback kiểm tra session ở tầng API để đảm bảo tính toàn vẹn khi WebSocket bị gián đoạn
